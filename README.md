# Iroto Web v2.14

这是 Iroto Web v2.14：演奏界面点击画面切换 UI 显示/隐藏版。

## 1. 演奏中点击画面的逻辑调整

之前的逻辑是：

```text
点击照片区域 → 显示 UI → 等几秒自动隐藏
```

v2.14 改成更接近一般视频 App 的逻辑：

```text
UI 隐藏时点击照片区域 → 显示 UI，并几秒后自动隐藏
UI 显示时再次点击照片空白处 → 立即隐藏 UI
```

这样不需要等待自动隐藏。

## 2. 不影响按钮操作

点击顶部/底部 UI 按钮不会触发这个画面点击逻辑。  
这个逻辑只作用于照片 / canvas 区域。

## 3. 版本号更新

资源路径更新为：

```text
app.js?v=2.14.0
styles.css?v=2.14.0
manifest.webmanifest?v=2.14.0
```

## 4. 保持不变

- v1.6 的矩阵相对旋转传感器逻辑保持不变
- 自动全屏保持 v2.13
- 底部浏览器推荐和权限提示保持 v2.13
- 拍照功能继续移除，只保留照片导入
- 独立 1080p 录制画布保持
- WebM 时长元数据修正保持

# v2.14.3 追加内容

这是基于重新上传的 v2.14.1 制作的无自动全屏版。  
没有修改传感器、准星平滑、加速度、照片显示、录制画布等手感相关逻辑。

## 1. 取消自动全屏

v2.14.3 不再调用：

```text
requestFullscreen()
```

也不再做 v2.14.2 的“保持全屏 / 自动恢复全屏”逻辑。  
这样可以避免全屏提示弹窗，以及某些浏览器反复进入/退出全屏的横跳问题。

## 2. 同时取消自动方向锁定

部分移动浏览器的 `screen.orientation.lock()` 需要全屏环境。  
为了避免它间接引发全屏相关问题，v2.14.3 也不再主动锁定屏幕方向。

```text

styles.css?v=2.14.3
manifest.webmanifest?v=2.14.3
```

# v2.14.4 追加内容

这是基于 v2.14.3 的修正版。

## 1. 不自动全屏，但恢复演奏时方向锁定

v2.14.3 取消自动全屏时，也取消了自动方向锁定，这是不合适的。  
v2.14.4 修正为：

```text
自动全屏：关闭
演奏开始时方向锁定：开启尝试
返回 / 停止后方向解锁：保持
```

也就是说，开始演奏时会尝试锁定当前方向：

```text
横屏开始 → 尝试锁定横屏
竖屏开始 → 尝试锁定竖屏
```

## 2. 浏览器机制限制

部分移动浏览器要求必须先进入全屏，才能调用 `screen.orientation.lock()`。  
由于 v2.14.4 不再自动全屏，如果浏览器拒绝方向锁定，会静默失败，不会影响演奏。

## 3. 版本号

资源路径更新为：

```text

styles.css?v=2.14.4
manifest.webmanifest?v=2.14.4
```

```text
v2.14.4
```
