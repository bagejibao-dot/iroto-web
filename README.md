# Iroto Web v2.4

这是 Iroto Web v2.4：WebM 时长元数据修正 + 横屏 safe-area 视觉居中版。

## 1. 修正手机相册里视频长度显示为 1 秒的问题

部分 Android / Chrome 的 MediaRecorder 导出 WebM 时，视频本身可以正常播放，但文件里的 Duration 元数据可能缺失或不准确。  
手机相册读取不到正确时长时，就可能显示成 1 秒。

v2.4 做了两点处理：

```text
录制停止时记录真实录制经过时间
保存前尝试把该时间写入 WebM 的 Duration 元数据
```

如果浏览器输出的是 MP4，则保留浏览器自身的 metadata，不强行改写。

注意：不同手机相册 App 对 WebM 支持不完全一致，因此该修正会明显改善兼容性，但不保证所有相册都能正确读取 WebM 时长。

## 2. 横屏 UI 改为按可显示区域居中

有些手机横屏时会避开前置摄像头 / 刘海区域，导致实际可显示区域不是完整屏幕宽度。  
v2.4 用 safe-area 重新计算横屏 UI：

```text
顶部标题按可显示区域居中
中间播放 / 停止按钮按可显示区域居中
底部 BPM 组按可显示区域居中
右侧兼容性 / 录制 / 回正按钮按同一可显示区域右轴线对齐
左侧语言 / 替换照片 / 计时器按同一可显示区域左轴线对齐
```

## 3. 保持不变

- v1.6 的矩阵相对旋转传感器逻辑保持不变
- 演奏界面继续保持照片原比例显示
- 录制继续使用独立照片比例的接近 1080p canvas
- 日语默认和中日英切换保持
- 保存信息多语言和删除二次确认保持
- Service Worker 仍然禁用，便于 GitHub Pages / Netlify 测试
